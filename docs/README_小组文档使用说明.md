## 四、Git贡献度佐证说明

### 4.1 五位组员独立Git目录

本小组采用**模块化分目录独立开发**模式，每位组员拥有专属Git目录，代码贡献可通过 `git log -- <directory>` 精确追溯：

| 组员 | 角色 | 专属Git目录 | 代码量 | 提交记录 |
|------|------|-----------|--------|---------|
| 组员1 | 核心引擎开发 | `src/main/java/com/iso11820/core/` | 3,026 行 | 可追溯 |
| 组员2 | 数据持久层开发 | `src/main/java/com/iso11820/entity/` `dao/` `sql/` | 3,327 行 | 可追溯 |
| 组员3 | UI可视化开发 | `src/main/java/com/iso11820/ui/` `resources/fxml/` | 1,177 行 | 可追溯 |
| 组员4 | 工具服务开发 | `src/main/java/com/project/service/` `utils/` | 4,631 行 | 可追溯 |
| 组员5 | 质量测试与交付 | `src/test/java/` `docs/` | 3,948 行 | 可追溯 |

### 4.2 贡献度查询命令

```bash
# 查看组员1 核心引擎模块的代码贡献
git log --author="组员1" -- src/main/java/com/iso11820/core/

# 查看组员2 数据持久层的代码贡献
git log --author="组员2" -- src/main/java/com/iso11820/dao/ src/main/java/com/iso11820/entity/ sql/

# 查看组员3 UI模块的代码贡献
git log --author="组员3" -- src/main/java/com/iso11820/ui/ src/main/resources/fxml/

# 查看组员4 工具服务层的代码贡献
git log --author="组员4" -- src/main/java/com/project/

# 查看组员5 测试与文档的代码贡献
git log --author="组员5" -- src/test/ docs/

# 统计各目录代码行数
git ls-files src/main/java/com/iso11820/core/ | xargs wc -l
```

### 4.3 开发过程证明

- **Git提交记录**：8次提交，覆盖项目初始化 → 核心引擎 → 数据持久层 → UI → 工具服务层 → 质量测试 → 文档撰写全流程
- **零合并冲突**：5人分目录独立开发，8次提交，0次合并冲突
- **AI辅助记录**：全组5人统一使用Claude Code辅助开发，代码注释风格一致
